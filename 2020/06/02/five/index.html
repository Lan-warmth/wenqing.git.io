下面列举一些常用例句：

创建数据库、数据表
– 创建数据库
create database 库名 charset=utf8;

– 使用数据库（使用后 才能进行创建表等操作）
use 库名;

– 创建表
create table 表名(
id int unsigned primary key auto_increment not null,
name varchar(20) default ‘’,
age tinyint unsigned default 0,
height decimal(5,2),
gender enum(‘男’,’女’,’中性’,’保密’) default ‘保密’,
cls_id int unsigned default 0,
is_delete bit default 0
);

说明：
name 称之为列名 ，varchar() 称之为列类型,以次类推
primary key：主键
auto_increment 自增长（1,2,3…）
not null：非空
unique: 唯一约束性
default ‘’ :默认为
decimal:小数，5为总长度，2为小数点后保留几位
enum：枚举类型

向表中插入数据
一条：insert into students values(1,’小明’,18,180.00,2,1,0)；
多条：insert into students values(1,’小明’,18,180.00,2,1,0),(0,’小明’,18,180.00,2,1,0);
tips：中文，日期用引号包起来,id只用填入第一个数据，后面的填0，会自动增长

查询所有字段

select * from 表名;

查询指定字段

select 列名1,列名2,… from 表名;

条件
使用where子句对表中的数据筛选，结果为true的行会出现在结果集中
例：select * from 表名 where 条件;
select * from students where id=1;

比较运算符
等于: =
大于: >
大于等于: >=
小于: <
小于等于: <=
不等于: != 或 <>

例：查询编号大于3的学生
select * from students where id > 3;

例：查询姓名不是“黄蓉”的学生
select * from students where id <= 4;

where 和having 区别

“Where” 是一个约束声明，使用Where来约束来之数据库的数据，Where是在结果返回之前起作用的，且Where中不能使用聚合函数。

“having”是一个过滤声明，是在查询返回结果集以后对查询结果进行的过滤操作，在Having中可以使用聚合函数。
having子句可以让我们筛选成组后的各组数据，where子句在聚合前先筛选记录．也就是说作用在group by子句和having子句前；而 having子句在聚合后对组记录进行筛选。

逻辑运算符
and
or
not

例：查询编号大于3的女同学
select * from students where id > 3 and gender=0;
例：查询编号小于4或没被删除的学生
select * from students where id < 4 or is_delete=0;

模糊查询
like
%表示任意多个任意字符
_表示一个任意字符

例：查询姓兰的学生
select * from students where name like ‘兰%’;
例8：查询姓兰并且“名”是一个字的学生
select * from students where name like ‘兰_’;
例9：查询姓兰或叫宁的学生
select * from students where name like ‘兰%’ or name like ‘%宁’;

范围查询
in
表示在一个非连续的范围内

例：查询编号是1或3或8的学生
select * from students where id in(1,3,8);

between … and …
表示在一个连续的范围内

例：查询编号为3至8的学生
select * from students where id between 3 and 8;

空判断
is null
判空

例：查询没有填写身高的学生
select * from students where height is null;

is not null
判非空

例：查询填写了身高的学生
select * from students where height is not null;

排序
order by
排序（默认从小到大 asc）
select * from 表名 order by 列1 asc|desc [,列2 asc|desc,…]

asc从小到大排列，即升序
desc从大到小排序，即降序

聚合函数

总数：count()表示计算总行数，括号中写星与列名，结果是相同的
例：查询学生总数
select count() from students;

最大值：max(列)表示求此列的最大值
例：查询女生的编号最大值
select max(id) from students where gender=2;

最小值：min(列)表示求此列的最小值
例：查询未删除的学生最小编号
select min(id) from students where is_delete=0;

求和：sum(列)表示求此列的和
例：查询男生的总年龄
select sum(age) from students where gender=1;

平均值：avg(列)表示求此列的平均值
例：查询未删除女生的编号平均值
select avg(id) from students where is_delete=0 and gender=2;

分组
group by
group by的含义:将查询结果按照1个或多个字段进行分组，字段值相同的为一组
group by可用于单个字段分组，也可用于多个字段分组

例：按性别分
select gender from students group by gender;

group by + group_concat()
1.group_concat(字段名)可以作为一个输出字段来使用，
2.表示分组之后，根据分组结果，使用group_concat()来放置每一组的某字段的值的集合

分页
limit start,count
从start(第几条)开始，获取count条数据
select * from 表名 limit start,count

例：查询前3行男生信息
select * from students where gender=1 limit 0,3;

连接查询
当查询结果的列来源于多张表时，需要将多张表连接成一个大的数据集，再选择合适的列返回

inner join-内连接查询：查询的结果为两个表匹配到的数据
left join-左连接查询：查询的结果为两个表匹配到的数据，左表特有的数据，对于右表中不存在的数据使用null填充
reght-join-右连接查询：查询的结果为两个表匹配到的数据，右表特有的数据，对于左表中不存在的数据使用null填充
通用语法：select * from 表1 inner或left或right join 表2 on 表1.列 = 表2.列

例：使用内/左/右连接查询班级表与学生表
select * from students inner/left/right join classes on students.cls_id = classes.id;

子查询

在一个 select查询 语句中,嵌入了另外一个 select查询 语句, 那么被嵌入的 select 语句称之为子查询语句

1.子查询分类

标量子查询: 子查询返回的结果是一个数据(一行一列)
例：先查询班级学生平均年龄，在查询大于平均年龄的学生。最终：查询班级学生的平均身高
select * from students where age > (select avg(age) from students);

列子查询: 返回的结果是一列(一列多行)
例：1.找出学生表中所有的班级 id; 2.找出班级表中对应的名字.最终：查询还有学生在班的所有班级名字
select name from classes where id in (select cls_id from students);

行子查询: 返回的结果是一行(一行多列)
需求: 查找班级年龄最大,身高最高的学生
行元素: 将多个字段合成一个行元素,在行级子查询中会使用到行元素
select * from students where (height,age) = (select max(height),max(age) from students);
